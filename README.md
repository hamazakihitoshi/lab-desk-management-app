# 実験台管理アプリ

## 概要
研究室における実験台不足という課題を解決するため、  
実験台の使用状況を可視化し、利用履歴をもとに混雑傾向を分析できるWebアプリを開発しました。

---

## 背景・課題
所属していた研究室では、実験台が8台しかない一方でメンバーが10人おり、  
同時に実験を行えない状況が発生していました。

また、  
- 誰が使っているか分からない  
- いつ空くのか分からない  
- 混雑する時間帯が把握できない  

といった問題がありました。

---

## 機能

### ① 実験台の使用管理
- 実験台ごとの使用状況をリアルタイムで表示
- 使用開始 / 終了の登録が可能

### ② 使用履歴の記録
- 利用者・開始時間・終了時間を記録
- 利用時間を自動計算

### ③ 混雑分析機能
- 曜日 × 時間ごとの利用データを集計
- 混雑 / やや混雑 を判定
- 連続する時間帯をまとめて表示

例：
- 月曜 14時〜17時ごろ混雑
- 火曜 10時〜12時ごろやや混雑

### ④ ダッシュボード表示
- 使用中台数
- 空き台数
- 平均利用時間
- 履歴件数

---

## 技術スタック

- Python（Flask）
- SQLite
- HTML / CSS
- matplotlib（グラフ可視化）

---

## 工夫した点

### ① 課題解決を意識した設計
単なる記録アプリではなく、  
「混雑を避けるための判断材料を提供する」ことを目的に設計しました。

---

### ② データ分析機能の実装
利用履歴を蓄積し、  
曜日・時間帯ごとの利用傾向を分析できるようにしました。

---

### ③ 視覚的に分かりやすいUI
- 使用中 / 空きを色で表示
- 混雑度をカード形式で表示
- ダッシュボードで重要情報をまとめて表示

---

### ④ 拡張性を意識した構成
- Flaskのルーティング、DB処理、ロジックを分離
- 将来的な機能追加を考慮した設計

---

## 動作確認について
実運用には至っていませんが、  
直近の利用状況をもとにデータを入力し、  
実際の利用を想定した形で動作検証を行いました。
